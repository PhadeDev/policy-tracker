New 2026-07-06 — abbreviate the long labels
Fixed 2026-07-06 — counts weren't respecting the selected tab
Switch()/If() used inside a Filter()'s predicate against a SharePoint data source can silently break delegation — the exact same conditions that work fine when Switch picks between whole separate Filter() calls (which is what HeaderGallery.Items itself already does) can give wrong/inconsistent results when the Switch is embedded as one term inside a single Filter's boolean expression instead. Confirmed live: chip counts summed to 25 while the actual "Issues" tab showed 0 items, which is only possible if the count formula wasn't really gating on the selected tab at all. Fix: expand each tab's condition inline with &&/|| instead of Switch, same shape as every other Filter in this app.
