Fix 1 — the four tabs read as four separate buttons, not one bar

Screen: Main. This is the biggest single difference from the prototype. Right now every one of DueNextWeek/FutureReleases/ItemIssues/History has its own full BorderThickness=1 and its own corner radius — four bordered boxes sitting next to each other with gaps between them. The prototype instead uses one shared rounded bar with the tabs as borderless segments inside it, so the selection state is the only thing that changes, not four independent boxes competing for attention.

Fix 3 — the Newsletter icon button is still solid blue

Button4 (the Newsletter/feedback icon, staff-only) is a ModernButton@1.0.0 with no explicit colour properties set — it's rendering with its default Fluent theme fill, which is why it's still solid blue after the other three nav buttons got the ghost-style recolour. This app family has already had trouble getting ModernButton to behave predictably for exactly this kind of light/outline styling (documented in the Bible), so rather than fighting its Appearance enum, replace it with a Classic/Button@2.2.0 icon-only button matching the other three exactly.

Fix 4 — the actual cause of the big gap under the chip row

Found it. HeaderGallery (the outer gallery, direct child of GalleryContainer) has:

Y: 48

That's a fixed 48px of empty space at the very top of the gallery before the first date group even starts rendering — on top of LeftSide's own LayoutGap=10 between the chip row and the gallery. Combined, that's roughly 58px of dead space before any content, which reads exactly like the gap in your screenshot.

ControlPropertyChange
HeaderGalleryY48 → 8
This Y=48 looks like a leftover from an earlier version of the screen that had more header content above the gallery. If dropping it to 8 causes the very first date group's header banner to sit too close to the chip row above it, nudge it up slightly (12–16) rather than back to 48.
