Part 1 — Release date bug fix

Screen: ViewItem. Root cause: the date picker's own OnChange was calling Reset(Self) whenever a non-Thursday was clicked. Since SelectedDate is bound live to varSelectedRelease.'Planned Publish Date', Reset(Self) snapped it straight back to the old/blank value — reads as "randomly clears itself" but happens on every non-Thursday click.

ControlPropertyChange
dpkPublishDate1OnChangeRemove the Reset(Self) line entirely, keep the Notify
Both Save buttons (top bar and bottom bar)OnSelect — the varErrPublishDate lineAdd a Weekday check so a non-Thursday date can no longer be saved at all
// dpkPublishDate1.OnChange — full replacement
If(
    Weekday(Self.SelectedDate, StartOfWeek.Monday) <> 4,
    Notify("Only Thursdays are allowed - pick a different date before saving.", NotificationType.Error)
);
Set(varUnsavedChanges, true);
Set(varErrPublishDate, false)
// Save button OnSelect — replace this one line (appears in both Save buttons)
Set(varErrPublishDate, IsBlank(dpkPublishDate1.SelectedDate) || Weekday(dpkPublishDate1.SelectedDate, StartOfWeek.Monday) <> 4);

Part 2 — Type filter chips

Step 1 — new variable

Step 2 — new container

In Main's Tree view, open LeftSide. From the Insert panel, insert a Horizontal container (not a plain Container, not Vertical) as the first child, directly above GalleryContainer. Name it TypeFilterBar.

Direction is fixed by which of the three container types you pick at insert time — there's no LayoutDirection property to set afterward, and no LayoutGap formula property either. Picking "Horizontal container" gives you the row layout directly. For spacing between chips, use the Gap stepper in the properties pane (next to the alignment icons when the container is selected) instead of a formula — set it to 8. Everything else below (Fill, Height, Width, padding, DropShadow) are normal formula properties, only direction and gap are insert-time/UI-only.

Why this can't publish "hundreds": ThisItem here is the date-group record (it has PlannedDate, ItemCount and GroupedItems — only the items sharing that one date), because this button sits inside the outer date-grouped gallery, not the inner per-card one. It's also gated to PlannedDate <= Today(), so it only shows up on a group that's actually due or overdue, and only if there's at least one unpublished item left in it.
