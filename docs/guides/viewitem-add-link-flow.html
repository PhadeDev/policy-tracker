ViewItem — Add Document Link: validation, auto-save, restyle

Replaces the "You must save this form before you can add links to documents" blocking message with one calm empty state that works whether the item has never been saved or has just been saved with no links yet — and gates link-adding on the three fields a link actually needs to exist (Title, Document Type, Planned Publish Date), auto-saving in the background instead of forcing a separate manual Save step first.

Why this is safe: a PolicyLinks row stores PolicyTrackerSource as a lookup to the parent's SharePoint ID, so a link genuinely cannot be created before the parent record has one. This guide doesn't remove that requirement — it just runs the same Patch the Save button already runs, automatically, at the moment the user asks to add a link, instead of making them click Save first and come back.

Step 7 — the actual validation + auto-save formula

This is the important part. Paste as OnSelect on the new btnAddLinkEmpty1:

// btnAddLinkEmpty1.OnSelect
=Set(varErrTitle, IsBlank(Trim(txtTitle1.Text)));
Set(varErrOwner, IsBlank(cmb_Owner1.Selected.Mail));
Set(varErrDocType, IsBlank(drpDocType1.Selected.Value));
Set(varErrPublishDate, IsBlank(dpkPublishDate1.SelectedDate));
If(
    varErrTitle || varErrOwner || varErrDocType || varErrPublishDate,
    Notify("Fill in Title, Document Type and Planned Publish Date before adding a link.", NotificationType.Warning),
    If(
        varParentSaved,
        Set(varSelectedLink, Defaults(PolicyLinks));
        Set(varShowLinkModal, true),
        Set(varSaving, true);
        Set(
            varSavedRecord,
            Patch(
                'Policy Proof Tracker',
                If(IsBlank(varSelectedRelease.ID), Defaults('Policy Proof Tracker'), varSelectedRelease),
                {
                    Title: Trim(txtTitle1.Text),
                    'Document Type': {Value: drpDocType1.Selected.Value},
                    Owner: {
                        Claims: "i:0#.f|membership|" & cmb_Owner1.Selected.Mail,
                        Department: "",
                        DisplayName: cmb_Owner1.Selected.DisplayName,
                        Email: cmb_Owner1.Selected.Mail,
                        JobTitle: "",
                        Picture: ""
                    },
                    'Planned Publish Date': dpkPublishDate1.SelectedDate,
                    'High Priority': ckbHighPriority1.Value,
                    'Reason for High Priority': txtReasonHighPri1.Text,
                    'External consultation required': ckbExtConsult1.Value,
                    'External consultation details': txtExtConsultDetails1.Text,
                    'Top Item': ckbTopItem1.Value,
                    'Title for Newsletter': txtNewsletterTitle1.Text,
                    Audience: txtAudience1.Text,
                    'Item summary': txtSummary1.Text,
                    Remarks: txtRemarks1.Text,
                    'Approved by Pillar Lead (Ready to Release)': ckbPillarLead1.Value,
                    'Approved For Release By': {Value: Coalesce(radApproval1.Selected.Value, "Not Yet Approved")},
                    Published: ckbPublished1.Value,
                    'Published Date': If(ckbPublished1.Value, Coalesce(dpkPublishedDate1.SelectedDate, Today()), Blank())
                }
            )
        );
        If(
            IsEmpty(Errors('Policy Proof Tracker')),
            Set(varSelectedRelease, varSavedRecord);
            Set(varSelectedItem, varSavedRecord);
            Set(varSelectPolicy, "Filled");
            Set(varOwnerDefault, Table({DisplayName: cmb_Owner1.Selected.DisplayName, Mail: cmb_Owner1.Selected.Mail}));
            Set(varParentSaved, true);
            Set(varUnsavedChanges, false);
            ClearCollect(colUnpublishedPolicies, Filter('Policy Proof Tracker', Published = false));
            ClearCollect(colLinks, Filter(PolicyLinks, PolicyTrackerSource.Id = varSavedRecord.ID));
            Set(varSaving, false);
            Notify("Saved - you can now add links.", NotificationType.Success);
            Set(varSelectedLink, Defaults(PolicyLinks));
            Set(varShowLinkModal, true),
            Set(varSaving, false);
            Notify("Save failed. Please check your connection and try again.", NotificationType.Error)
        )
    )
)
The Patch block above is copied verbatim from btnSaveClose1.OnSelect so it writes exactly the same fields the real Save button does — deliberately duplicated rather than factored out, matching how this screen already duplicates the same block between btnSaveClose1 and btnSave1. If you ever change what Save writes, update this copy too.
Owner is included in the required-field check even though you only asked for Title, Document Type and Planned Publish Date, because Owner is a genuine field on the Patch call and ViewItem's OnVisible already defaults it to you automatically — in practice it's essentially never the thing blocking someone, it's just there so the auto-save can't silently write a blank Owner. The on-screen hint text in Step 5 deliberately only lists the three you asked about, so nobody sees "Owner" called out for a field they never had to think about. If you'd rather drop Owner from the check entirely, remove the || varErrOwner from the If and the Set(varErrOwner, ...) line.

Step 8 — same guard on the "+" button that appears once links already exist

btnNewLink_1 (the floating round "+" in the corner, only visible once colLinks already has at least one row) currently opens the link modal with no validation at all. By definition the parent is already saved by the time this button can show, so it only needs the validation half, not the auto-save branch:

// btnNewLink_1.OnSelect — full replacement
=Set(varErrTitle, IsBlank(Trim(txtTitle1.Text)));
Set(varErrOwner, IsBlank(cmb_Owner1.Selected.Mail));
Set(varErrDocType, IsBlank(drpDocType1.Selected.Value));
Set(varErrPublishDate, IsBlank(dpkPublishDate1.SelectedDate));
If(
    varErrTitle || varErrOwner || varErrDocType || varErrPublishDate,
    Notify("Fill in Title, Document Type and Planned Publish Date before adding a link.", NotificationType.Warning),
    Set(varSelectedLink, Defaults(PolicyLinks));
    Set(varShowLinkModal, true)
)

Result

  • Brand-new item, required fields still blank: empty-state panel shows the muted "No document links yet" heading, the red hint line naming exactly what's missing, and the Add Document Link pill button — clicking it just shows the same warning toast, no confusing full-screen block.
  • Brand-new item, Title/Document Type/Planned Publish Date filled in: clicking Add Document Link saves the record in the background (same as clicking Save) and opens the link editor in one step — no separate manual save, no "you must save first" message ever seen.
  • Existing item with no links yet: identical empty state and identical validation, just skips straight to opening the modal since varParentSaved is already true.
  • Existing item with links already: the "+" FAB now respects the same three required fields before letting you add another one, closing the gap where it previously had no validation at all.
