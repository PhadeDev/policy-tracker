Main In-Place Redesign Guide

This guide changes the existing Main screen gradually to match the successful Main_1.yaml prototype. It avoids repasting a full screen and focuses on replacing sections in place.

Source of truth: yaml/Main_1.yaml. The approval stepper, completion mint, anomaly amber, hero progress bars, Release Pipeline widget, and copy-link formula are already proven there.

Final Checks

  • No rounded Label or rounded Rectangle controls.
  • No conditional ModernButton.BasePaletteColor; use Classic Button for tabs and card click overlays.
  • Progress@1.1.34 uses string style values: "Success", "Rounded", "Large".
  • Card click target sits behind labels/actions but above the background.
  • Stepper lines follow the dot they lead from.
