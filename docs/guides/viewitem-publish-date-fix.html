ViewItem — Planned Publish Date showing "2 Jul26 2026"

Screen: ViewItem, control dpkPublishDate1. Not a format string typo — the control itself is wrong. It's currently ModernDatePicker@1.0.1, not DatePicker@0.0.46 — the exact control already blacklisted in the Bible for a different bug (a SelectedDate authoring error). Looks like this field got rebuilt by hand at some point and "Modern Date picker" got picked from the Insert panel instead of the plain "Date picker." ModernDatePicker's Format property doesn't behave like a free-text formula the way the classic one does, which is almost certainly why it renders that squashed "2 Jul26 2026" instead of respecting the format string.

This is a full control swap, not a property tweak. Delete the existing dpkPublishDate1 and paste this in as a new control with the same name.
- dpkPublishDate1:
    Control: DatePicker@0.0.46
    Properties:
      BorderColor: =If(!IsBlank(Self.SelectedDate) && Weekday(Self.SelectedDate, StartOfWeek.Monday) <> 4, Color.Red, RGBA(223, 229, 224, 1))
      BorderStyle: =BorderStyle.Solid
      BorderThickness: =1
      Fill: =RGBA(255, 255, 255, 1)
      Font: =Font.'Open Sans'
      FontSize: =14
      Format: =Text(Self.SelectedDate, "d mmmm yyyy")
      Height: =40
      OnChange: |-
        =If(
            Weekday(Self.SelectedDate, StartOfWeek.Monday) <> 4,
            Notify("Only Thursdays are allowed - pick a different date before saving.", NotificationType.Error)
        );
        Set(varUnsavedChanges, true);
        Set(varErrPublishDate, false)
      SelectedDate: =varSelectedRelease.'Planned Publish Date'
      StartOfWeek: ="Monday"
      Width: =Parent.Width
      Y: =22

Two syntax differences from what you had

PropertyWhy it changed
SelectedDate replaces DefaultDateThis control's actual settable property is different from ModernDatePicker's.
StartOfWeek: ="Monday"Plain quoted string on this control — not the StartOfWeek.Monday enum, which is specific to ModernDatePicker and will error here.

Also dropped FontWeight: ="" and Placeholder — neither is a real property on DatePicker@0.0.46, they were just silently accepted/ignored by ModernDatePicker.
