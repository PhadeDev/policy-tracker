ViewItem — Planned Publish Date showing "2 Jul26 2026"

Correction: the first version of this page told you to swap the whole control out for DatePicker@0.0.46. That was unnecessary and wrong — you found the real fix yourself before that guidance was even useful, and it's much simpler.

Screen: ViewItem, control dpkPublishDate1 (ModernDatePicker@1.0.1). The actual cause: ModernDatePicker.Format does not support the 4-m "mmmm" token (full month name) the way DatePicker@0.0.46.Format does. With four ms it silently mis-renders — no error, just garbled output like "2 Jul26 2026" — instead of the expected "2 July 2026".

The fix: drop to three ms. The exact same control, no swap needed at all.
// dpkPublishDate1.Format
Text(Self.SelectedDate, "d mmm yyyy")

That renders "2 Jul 2026". Everything else on the control — DefaultDate, the Thursday-only OnChange check, BorderColor, all of it — stays exactly as it was. Just the one property, just those four characters shortened to three.
