Loading Screen — Branded App-Start Splash

Replaces StartScreen's single bare "Loading..." label with a real branded splash. Component: cmpLoadingScreen from powerappsui.com, archived at yaml/components/LoadingScreen/.

One honest limit on what this can actually do: Power Apps runs App.OnStart completely before showing any screen — there's no way to display custom UI while OnStart itself is still running, the platform's own generic loading indicator covers that part regardless of what we build. So LoadingSteps here can't be wired to genuine live progress of your real OnStart (building colChoiceColors, etc.) — that's already finished by the time this screen even renders. What this guide gives you instead is honest: a deliberate, short, branded pause (controlled entirely by MinDisplayTime, which you set) with cosmetic step labels, rather than a technically-impossible "real progress bar." If you want true multi-step progress tracking later, that would mean moving the collection-building out of App.OnStart and into this screen's own OnVisible instead — a bigger change, not in scope here.

1. Verify the component pastes cleanly

Insert → Custom → New Component, paste the full contents of yaml/components/LoadingScreen/LoadingScreen.yaml, name it cmpLoadingScreen.

Uses Progress@1.1.34, not yet confirmed live anywhere in this app. Standard modern control, low risk, but confirm the import renders correctly first.
Test: component imports clean, previewing it directly shows the logo mark, wordmark, and progress dots without errors.

2. Change what StartScreen.OnVisible does

Currently it navigates immediately on load, which is exactly why the old label never really had time to show. Replace the whole property with:

=Set(
    varStartDestination,
    If(!IsBlank(Param("ID")), "ViewItem", "Main")
)

This is the same logic as before — deep link present means go to ViewItem, otherwise Main — just stored instead of acted on immediately. The actual Navigate() moves to the component's OnReady event below, so the splash gets a real, deliberate moment on screen instead of flashing for a single frame.

3. Place the component, wire OnReady

Drag one instance of cmpLoadingScreen from the Components panel onto StartScreen, delete the old lblLoading label, and set:

Height: =App.Height
Width: =App.Width
X: =0
Y: =0
Theme: ="Light"
AccentColor: =RGBA(11, 74, 54, 1)
LogoLetter: ="P"
AppName: ="Policy"
AppNameAccent: =" Tracker"
AppSubtitle: ="Cadet Forces Document Tracking"
CompanyName: ="Policy Tracker"
AppVersion: ="V2"
LoadingSteps: =Table(
    {msg: "Loading colour palette…"},
    {msg: "Loading document list…"},
    {msg: "Checking staff access…"}
)
CurrentStep: =3
MinDisplayTime: =400
ErrorMessage: =""
OnReady: =Navigate(varStartDestination, ScreenTransition.None)

CurrentStep: =3 is set fixed at "all steps shown as done" rather than tracked live, for the honest reason above — there's nothing left running by this point to track. MinDisplayTime: =400 is short on purpose, matching what you asked for: enough to register as an intentional brand moment, nowhere near long enough to feel like a wait. Nudge it up a little (600–800) if 400ms reads as too much of a flash once you see it live — that's a one-number change, not a redesign.

Test: reload the app fresh (not just navigating within it) and confirm the splash appears briefly, shows your branding, then lands on Main. Then test the deep-link path directly (open the app with ?ID= in the URL, same method used to test the deep-link "not found" case earlier) and confirm it lands on ViewItem instead.
