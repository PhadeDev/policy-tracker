"No releases" message didn't account for Future weeks

Main · property edit · no risk

lblNoReleasesMain only ever checked for the Historic tab, so it showed "No releases this week" for both the Current tab and the Future tab — wrong for Future, since it's not "this week" at all. There are actually four tabs (Current, Future, UnapprovedHistoric, Historic), so this now branches properly across all of them.

  1. Open Main.
  2. In the Tree view, select lblNoReleasesMain (inside the weekly release group's gallery template).
  3. Select Text in the Properties dropdown, select all, and replace with:
Switch(
    varSelectedState,
    "Historic", "No releases went out that week",
    "UnapprovedHistoric", "No releases went out that week",
    "Future", "No releases planned for that week",
    "No releases this week"
)
Test: check the message on each tab — Current says "No releases this week", Future says "No releases planned for that week", and both Historic/UnapprovedHistoric still say "No releases went out that week".
