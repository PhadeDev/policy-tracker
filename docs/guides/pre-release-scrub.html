Block saving personal OneDrive links as document links

ViewItem · property edit · no risk

The Open In App icon-scan already looks for the real SharePoint document-library domain (modgovuk.sharepoint.com) on OpenDesktop_2.Visible. A personal OneDrive share link (e.g. https://modgovuk-my.sharepoint.com/:w:/r/personal/...) uses a different domain — the -my.sharepoint.com "my site" domain, not the document library — so it was never going to be recognised, and worse, it's not something colleagues should be linking to from a tracked policy item anyway (personal OneDrive isn't shared storage). First attempt used a toast on Save, but toasts are easy to miss. Replaced it with a persistent inline text block under the URL field itself (reusing the existing error-label spot, so it shows live as soon as the link is typed/pasted, not just on Save). Save itself still re-checks the same condition independently as a fallback — it doesn't just rely on the label being visible, so even if the inline warning were somehow not showing, Save would still silently refuse to submit a personal OneDrive link.

  1. Open ViewItem.
  2. In the Tree view, find ErrorMessage29_2 (the small text under the "Direct URL to document" field).
  3. Select Text in the Properties dropdown, select all, and replace with:
If(
    "modgovuk-my.sharepoint.com" in Lower(DataCardValue29_2.Text),
    "This is a personal OneDrive link. Please save this document into SharePoint first, then paste the SharePoint link here.",
    frmLink_2.Valid = false && !IsBlank(frmLink_2.Error),
    "URL is required.",
    ""
)
  1. Still on ErrorMessage29_2, select Wrap in the Properties dropdown and set it to true (the message is longer than the old "URL is required." text, so it needs to wrap onto a second line instead of being cut off).
  2. Now find btnSaveLink_2 (the Save button in the same modal). This is the fallback check — it runs the same domain test again independently, so Save still refuses to submit even if the inline warning above wasn't showing.
  3. Select OnSelect in the Properties dropdown, select all, and replace with:
If(
    !varSubmitting && !("modgovuk-my.sharepoint.com" in Lower(DataCardValue29_2.Text)),
    Set(varSubmitting, true);
    SubmitForm(frmLink_2);
    // Do NOT refresh here
    Set(varSubmitting, false)
)
Test: paste a modgovuk-my.sharepoint.com/:w:/r/personal/... link into the URL field — the warning text should appear straight away under the field, wrapped onto two lines, and hitting Save should do nothing while it's showing. A normal modgovuk.sharepoint.com document-library link should show no message and still save as before.
