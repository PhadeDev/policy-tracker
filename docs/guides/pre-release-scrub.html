1. ViewItem — stale Owner/Release Date after visiting a blank entry The reported bug

This is a real regression, not a new bug — it's the exact same fault class as a bug fixed once already (cmb_Owner1 missing from the Reset() chain, commit 2a75945, "Find real cause of Owner bug"), except it's now missing again and a second control (dpkPublishDate1, the Planned Publish Date picker) has the identical gap and may never have been in the list at all.

Why this causes exactly what you saw: ViewItem.OnVisible explicitly Reset()s 14 input controls so their Default re-evaluates against whichever item you just opened — Reset() is the only thing that forces a control's Default to re-apply; without it, a control's displayed value just sits at whatever it last was. cmb_Owner1 and dpkPublishDate1 were both left out. So: open a blank new entry (Owner/Date picker sit however they render for a new item) → try to save without filling everything → close without saving → open a real item with real data → cmb_Owner1 and dpkPublishDate1 don't refresh, they carry over stale state from the blank entry. The "Fill in Title, Document Type and Planned Publish Date to add document links" red hint text near the link list is computed live from these exact controls' current values (not from the varErr* flags), so a stale/blank picker makes that warning show even though the record you're looking at is completely valid. Clicking Save re-reads the actual field values at that moment and clears it — which is exactly the "hit Save and it clears" behaviour you found as the only workaround.

Click on empty screen background (not any child control) so the Properties dropdown at the top of the canvas shows ViewItem as the selected object. In the formula bar's left dropdown (next to the fx icon), select OnVisible. Click into the formula bar itself, select all existing text (Ctrl+A) and delete it, then paste the block below.

4. Formulas — GetFileExt dedup, confirmed still not applied Tidy-up, zero behaviour change

Also still duplicated in the fresh export. Repeating for completeness.

Where to click: this one isn't on a screen — click the App entry at the very top of the Tree view (above all screens), then select Formulas from the Properties dropdown. This is one long text box holding every named formula in the app (BuildLinksForSource, RefreshActiveCache, etc.) — don't select-all here, since that would delete formulas this guide isn't touching. Instead, click into the box and paste the new GetFileExt formula below anywhere between two existing formulas (e.g. right after the closing }; of BuildLinksForSource).

Add this new formula:

GetFileExt(LinkUrl: Text): Text = With(
    {FileParamStart: Find("file=", LinkUrl, 1)},
    With(
        {
            FileNamePart: If(
                FileParamStart > 0,
                With(
                    {AmpPos: Find("&", LinkUrl, FileParamStart + 5)},
                    Mid(LinkUrl, FileParamStart + 5, If(AmpPos > 0, AmpPos - (FileParamStart + 5), Len(LinkUrl) - (FileParamStart + 5) + 1))
                ),
                If(Find("?", LinkUrl, 1) > 0, Left(LinkUrl, Find("?", LinkUrl, 1) - 1), LinkUrl)
            )
        },
        Switch(
            true,
            Right(FileNamePart, 5) = ".docx", "docx",
            Right(FileNamePart, 4) = ".doc", "doc",
            Right(FileNamePart, 5) = ".xlsx", "xlsx",
            Right(FileNamePart, 4) = ".xls", "xls",
            Right(FileNamePart, 5) = ".pptx", "pptx",
            Right(FileNamePart, 4) = ".ppt", "ppt",
            Right(FileNamePart, 4) = ".pdf", "pdf",
            ""
        )
    )
)

Then, still in that same App → Formulas box, find the existing BuildLinksForSource and RefreshActiveCache formulas (each starts with its name and ends with a closing };) and replace each one individually with its shortened version below — everything outside those two blocks stays exactly as it is:

BuildLinksForSource(SourceID: Number): Boolean = {
    ClearCollect(
        colLinks,
        AddColumns(
            Filter(PolicyLinks, PolicyTrackerSource.Id = SourceID),
            FileExt,
            GetFileExt(LinkUrl)
        )
    );
    Set(varLinksSourceID, SourceID);
    true
};
RefreshActiveCache(): Boolean = {
    ClearCollect(
        colActiveItems,
        Filter(
            'Policy Proof Tracker',
            'Planned Publish Date' >= Today() &&
            'Planned Publish Date' <= DateAdd(Today(), 28, "Days")
        )
    );
    ClearCollect(
        colActiveLinks,
        AddColumns(
            Filter(PolicyLinks, PolicyTrackerSource.Id in colActiveItems.ID),
            FileExt,
            GetFileExt(LinkUrl)
        )
    );
    Set(varActiveCacheReady, true);
    true
};
