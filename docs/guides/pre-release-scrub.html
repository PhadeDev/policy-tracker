Owner avatar picture missing/flaky on Main

Main · property edit · no risk

The owner picture on Main (UsrIMG) was bound to ThisItem.Owner.Picture — the raw SharePoint Person-column picture reference, which depends on SharePoint's own thumbnail/claims cache and is well known for returning blank on a fresh load or for some users regardless of retries. ViewItem never has this issue because it uses a different, reliable source: a direct Microsoft Graph call, Office365Users.UserPhotoV2(...). Switched Main to use the same Graph call instead of the SharePoint field.

  1. Open Main.
  2. In the Tree view, select UsrIMG (the round owner avatar image inside the release card gallery template).
  3. Select Image in the Properties dropdown, select all, and replace with:
If(!IsBlank(ThisItem.Owner.Email), Office365Users.UserPhotoV2(ThisItem.Owner.Email), "")
Test: scroll through the release list on Main and confirm every owner's photo now loads consistently (matching what already worked reliably on ViewItem), instead of sometimes appearing blank.
