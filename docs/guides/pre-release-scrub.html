Duplicate button visible on unsaved items

ViewItem · property edit · no risk

The Duplicate button (top-right, copy icon) had no Visible condition at all, so it showed even on a brand-new item that hasn't been saved yet — clicking it in that state isn't a tested path and shouldn't be reachable. Fixed by tying it to varParentSaved, the same flag the screen already uses elsewhere to mean "this item has actually been saved" (false on a new item, true once saved or opened from an existing record).

  1. Open ViewItem, open any item.
  2. In the Tree view, select btnDuplicate1 (the copy-icon button, top-right of the header).
  3. In the Properties dropdown, click + Add property and choose Visible (it isn't set at all right now).
  4. Set it to:
varParentSaved
Test: open a brand-new item (before saving) — the Duplicate button should be gone. Save it, or open any existing item — it should be back.
