1. Invalid formula in ViewItem.OnVisible — do this one first, before anything else TOP PRIORITY, LIKELY ROOT CAUSE OF THE OWNER BUG TOO

ViewItem.OnVisible currently contains an invalid Navigate(Main, ScreenTransition.Fade) call, sitting inside the deep-link "item not found" branch. Studio doesn't allow Navigate inside a screen's own OnVisible at all, even nested inside an If that only reaches it conditionally, and flags it as an error (the "Navigate cannot be used here" message from earlier). It still saves despite the error, but Power Fx compiles each property as one whole unit, so a hard error anywhere in OnVisible can mean the entire property fails to run reliably at runtime, not just the one broken branch.

This is very likely why the Reset(cmb_Owner1) fix hasn't worked even though it's genuinely sitting in the formula — if OnVisible as a whole isn't executing cleanly because of this error, nothing later in that same formula can be trusted either, including the Owner reset, until this is fixed. Do this one before re-testing Owner again.
