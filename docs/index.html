Policy Tracker V3

Built 24 Jun 2026

Canvas app — tracks ACCT UK policy documents through the release pipeline. Four tab views (Current / Future / Issues / Archive), date-grouped gallery, colour-coded document types, approval workflow, and deep-link support. Wraps the Policy Proof Tracker SharePoint list with a PolicyLinks child list for associated documents.

Before You Start

  1. Add the one new column to the SharePoint list (see below).
  2. Open Power Apps Studio → New blank canvas app (tablet landscape, 1366×768).
  3. Add both data connections listed below.
  4. Set App → OnStart using the code block below.
  5. Create 5 blank screens, then paste each YAML in order.

Data Connections

Add these in the Data panel before pasting any YAML:

  • SharePoint → connect to your SharePoint site → select Policy Proof Tracker
  • SharePoint → same site → select PolicyLinks
  • Office365Users → used for the owner people-picker on ViewItem

SharePoint List Changes

The existing Policy Proof Tracker list is used as-is. One new column must be added:

List: Policy Proof Tracker existing

This list already exists. Do not recreate it. Only add the one new column below.
Column nameTypeNotes
Top ItemYes/No (Boolean)ADD THIS Default: No. Used to flag entries that should appear first in the policy newsletter. The internal field name will be Top_x0020_Item.
All other columns—Leave as-is. The app uses internal column names directly from the schema.
Column name gotchas: Several columns have URL-encoded internal names. The app uses these exactly:
  • TrackerType — displays as "Document Type"
  • PublishDate — displays as "Planned Publish Date"
  • ActualPublishedDate — displays as "Published Date"
  • ReadyForRelease — displays as "Approved by Pillar Lead (Ready to Release)"
  • External_x0020_consultation_x002 — displays as "External consultation required"
  • Item_x0020_summary — displays as "Item summary"
  • Title_x0020_for_x0020_Newsletter — displays as "Title for Newsletter"
  • Approved_x0020_For_x0020_Release — displays as "Approved For Release By" (radio button)
  • Top_x0020_Item — displays as "Top Item" (the new column you are adding)
If any Patch calls fail with a field-not-found error, check the actual internal name in SharePoint list settings.

Paste Order

  1. Create 5 blank screens in Power Apps (they can be named anything — the YAML renames them on paste).
  2. Click the first blank screen, open the three-dot menu, choose Paste YAML.
  3. Repeat for each screen in order: StartScreen, Main, ViewItem, Historic, Overview.
  4. Set the startup screen to StartScreen in App → StartScreen.
Top Item SP column: The Top Item Yes/No column must exist in the Policy Proof Tracker list before you paste ViewItem. Without it, the TopItem_DataCard1 DataCard will throw a binding error on import. Add the column first, then refresh the SharePoint data source in the Data panel.
Thursday dates: The Duplicate modal only shows Thursdays as valid release dates. Thursdays within 10 days of today are hidden unless the entry is marked High Priority. If your organisation changes release cadence, update the colThursdays ClearCollect in App.OnStart accordingly.

What Changed in V3

  • New: "Owned by Me" toggle on Main — filters Current, Future, and Issues by Owner field (not creator)
  • New: Top Item flag — amber strip on gallery cards, amber DataCard in ViewItem newsletter section, ⭐ flag on Overview Kanban
  • New: Links always open in a new tab — users no longer navigate away from the app when clicking a document link
  • New: Scroll position is restored when returning from ViewItem to Main
  • Fixed: Stat card counts were using wrong date ranges — now correct
  • Fixed: varHistoricLimit was 30, now correctly 28 (4 weeks of Thursdays)
  • Fixed: gblMorphTimestamp was not initialised in OnStart — now set on app launch
  • Removed: Approved For Release By DataCard was incorrectly removed in an earlier draft — it is fully present in V3
  • Redesigned: Gallery cards rebuilt with Labels (not TextInput), type-colour left bar, watermark, owner avatar, AutoLayout pills
